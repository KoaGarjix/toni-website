# PLANERING.md

## Project Purpose

This project is the planning foundation for the personal website of Toni Vähäkuopus.

The website should be a personal home on the web. It should present Toni in a clean, professional and trustworthy way, while still leaving room for personal interests, technical hobbies, experiments and future notes.

Version 1 should be simple, stable and easy to maintain.

The website should not try to become a final dream site immediately.

## Core Direction

The site should be one main website, not several separate websites.

The homepage should give a professional overview.

Deeper pages may be more personal, technical and experimental.

The primary language is English.

A Swedish version may be considered later, but multilingual support is not part of Version 1.

## Tone

The tone should be clear, calm and personal without becoming too informal.

The homepage should feel professional and neutral.

The About page may be slightly more personal, but still clean.

Lab and future notes may have a more relaxed and experimental tone.

## Main Sections

The site should start with these sections:

Home

About

Projects

Lab

Contact

## Section Roles

Home gives a short introduction and previews selected Projects and Lab items.

About explains who Toni is, with a balanced mix of work background, technical interests and personal interests.

Projects contains curated and more presentable work, builds or ideas.

Lab contains experiments, drafts, learning notes, AI-assisted builds and unfinished work.

Contact gives simple ways to reach Toni.

## Content Strategy

Projects should be curated.

Lab can contain rougher and more experimental material.

The site should not be a pure CV site.

The site should not be a sales page.

The site should not be a company website.

The site should not be a full blog in Version 1.

The structure may prepare for notes or blogging later, but there is no requirement to publish regular posts in Version 1.

## Technical Direction

Use Astro for the main site.

Use Markdown for content where practical.

Keep the site static-first.

Avoid backend complexity in Version 1.

Avoid databases in Version 1.

Avoid CMS in Version 1.

Avoid unnecessary JavaScript.

Keep the site easy to deploy and easy to maintain.

## Design Direction

The design should be clean, fast and readable.

Mobile support matters from the beginning.

The design should not be overbuilt.

The first version should prioritize structure, readability and content over visual effects.

## Future Flexibility

Future advanced applications, dashboards or services do not have to be built inside this Astro site.

The main website can later link to separate apps, repositories or services if needed.

Version 1 should stay focused on the personal website foundation.

## AI Workflow

ChatGPT is used for planning, architecture, structure, file content and reasoning.

Codex in VS Code is used for local file creation and code edits.

Perplexity may be used only when current technical information needs to be checked.

Claude Code is not needed for the foundation phase.

Claude Code may be considered later for advanced components or more complex frontend work.

Gemini may be used later for text or SEO suggestions, but it should not define the core structure.

## AI Output Rules

AI should suggest the simplest working solution first.

AI should avoid unnecessary dependencies.

AI should provide exact file paths when creating or editing files.

AI should avoid pseudocode when real code is needed.

AI should not leave unfinished placeholders unless explicitly requested.

AI should keep the project deployable after major steps.

AI should explain technical decisions briefly and practically.

## Version 1 Scope

Version 1 should include:

Home page

About page

Projects page

Lab page

Contact page

Basic layout

Basic navigation

Markdown-friendly content structure

README.md

CHANGELOG.md

Version 1 should not include:

CMS

Database

Authentication

Advanced dashboard features

Interactive maps

Full blog system

Multilingual support

Complex animations

## Definition of Done for Version 1

The site can run locally.

The site can be deployed.

The homepage gives a clear overview.

The About page feels personal but clean.

Projects can show curated items.

Lab can show experimental items.

Contact information is easy to find.

The site works on mobile and desktop.

The repository includes basic documentation.

The project remains simple enough to understand and maintain.
